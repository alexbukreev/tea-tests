# Проект: Платформа коллективной дегустации чая  
MVP-план на три спринта (по 2 недели)

## Цель проекта

Создать онлайн-платформу, которая помогает проводить коллективные дегустации чая и собирать структурированные оценки участников.  
Платформа должна:

- позволять пользователям записываться на дегустации через Telegram-бота и/или веб;
- давать удобный инструмент для оценки чаёв (лучевые диаграммы);
- сохранять все данные на сервере для дальнейшего анализа и работы с ними;
- по итогам дегустации выдавать участникам простой, понятный отчёт и «профиль чаемана».

Работа разбита на три двухнедельных спринта. По результатам каждого спринта — рабочий, тестируемый функционал.

---

## Спринт 1 — Интерфейс оценки и демо на телефоне

**Основная цель:**  
Показать живой интерфейс дегустации на телефоне: форму с ~18 параметрами и лучевой диаграммой, которая сразу визуализирует вкусовой профиль чая.

Фокус этого спринта — **пользовательский интерфейс**, без сложной серверной логики. Данные могут быть тестовыми, без постоянного сохранения.

### Что планируется сделать

1. **Базовый фронтенд-проект**
   - Настройка проекта на React + Vite + shadcn/ui.
   - Базовая структура страниц и компонентов.

2. **Мобильный интерфейс дегустации**
   - Страница оценки чая, оптимизированная под телефон:
     - фиксированная лучевая диаграмма в верхней части экрана;
     - список параметров (около 18-ти) под диаграммой;
     - управление параметрами через ползунки (слайдеры).
   - Диаграмма **всегда видна**, значения на ней автоматически обновляются при изменении ползунков.

3. **Поведение диаграммы**
   - Базовый вариант: диаграмма не интерактивна, служит визуальной обратной связью по введённым значениям.
   - Если позволяет бюджет и время — добавляется опция изменения значений прямо по точкам на диаграмме (перетаскиванием по радиальным осям).

4. **Демо-сценарий**
   - Подготовка демонстрационного сценария:
     - один «виртуальный» образец чая;
     - заполнение параметров и визуализация профиля;
     - возможность показать интерфейс на реальном телефоне.

### Результат спринта 1

- Живой интерфейс дегустации, который можно открыть на телефоне:
  - ползунки для параметров,
  - лучевая диаграмма, отражающая текущие значения.
- Возможность провести «сухую» демонстрацию процесса оценки без Telegram и без постоянного сохранения данных.

## Спринт 2 — Бэкенд, Telegram и сохранение оценок

**Основная цель:**  
Подключить интерфейс к реальному серверу: регистрация через Telegram, создание дегустаций и образцов чая, сохранение оценок в базе данных.

### Что планируется сделать

1. **Архитектура бэкенда и база данных**
   - Настройка серверной части (Node.js).
   - Определение основных сущностей:
     - пользователи,
     - дегустации,
     - образцы чая,
     - параметры оценок,
     - оценки (результаты).
   - Настройка базы данных (PostgreSQL) и миграций.

2. **Интеграция с Telegram**
   - Подключение Telegram-бота к серверу.
   - Регистрация пользователей по их Telegram-аккаунту:
     - при первом `/start` пользователь создаётся в системе.
   - Механика персональных ссылок:
     - бот выдаёт пользователю ссылку на форму оценки,
     - переход по ссылке автоматически идентифицирует пользователя без логина/пароля.

3. **Связка интерфейса и бэкенда**
   - Интерфейс дегустации из Спринта 1 начинает работать с реальными данными:
     - загрузка списка параметров и образцов чая с сервера;
     - отправка оценок на сервер и сохранение в базе.
   - Возможность провести настоящую дегустацию с небольшой группой:
     - регистрация через Telegram,
     - заполнение форм,
     - фиксация данных.

4. **Базовый сервис для организатора**
   - Возможность создать дегустацию и список чаёв (на этом этапе — через служебный интерфейс администратора или простые технические средства).
   - Проверка, что данные по дегустации корректно сохраняются и доступны для дальнейшей обработки.

### Результат спринта 2

- Полный рабочий цикл дегустации:
  - пользователь регистрируется через Telegram;
  - получает персональную ссылку;
  - заполняет параметры в интерфейсе с лучевой диаграммой;
  - оценки сохраняются на сервере и привязаны к пользователю/чаю/дегустации.
- Подготовленная структура данных для будущих отчётов и аналитики.

## Спринт 3 — Отчёты для участников, админ-панель и интеграция с Airtable

**Основная цель:**  
Дать участнику понятный отчёт о его вкусовом профиле и сравнении с группой, а организатору — удобный способ управлять справочниками (дегустации, чаи, описания) через Airtable.

### Что планируется сделать

1. **Отчёт для участника**
   - Формирование профиля участника на основе его оценок:
     - какие характеристики он выделяет;
     - какие типы чаёв ему ближе.
   - Сравнение с группой:
     - лучевая диаграмма «участник vs средние оценки группы»;
     - текстовые выводы (например: «вы предпочитаете более сладкие и менее горькие чаи, чем средний участник этой дегустации»).
   - Ссылка на отчёт отправляется пользователю через Telegram по завершении дегустации.

2. **Интеграция с Airtable (управление справочниками)**
   - Настройка базы в Airtable с таблицами:
     - дегустации (название, дата, описание),
     - образцы чая (название, примечания, порядок в дегустации),
     - при необходимости — текстовые описания профилей.
   - Бэкенд подключается к Airtable через API и:
     - читает списки дегустаций и чаёв из Airtable;
     - использует эти данные в интерфейсе и в отчётах.
   - Организатор управляет содержимым (списки чаёв, тексты и т.п.) прямо в Airtable, без участия разработчика.

3. **Минимальный админ-интерфейс**
   - Защищённый вход для организатора (через Telegram и служебную ссылку).
   - Просмотр состояния дегустаций:
     - сколько участников,
     - кто завершил заполнение оценок.
   - Кнопка/действие для отправки участникам ссылок на их отчёты.

### Результат спринта 3

- Участник получает персональный отчёт (диаграмма + текстовый профиль, сравнение с группой).
- Организатор управляет списком дегустаций и чаёв через удобные табличные формы в Airtable.
- Внутри сервиса есть минимальная админ-панель для контроля хода дегустаций и запуска рассылки результатов.
