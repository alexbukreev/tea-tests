# Проект: Платформа коллективной дегустации чая  
MVP-план на три спринта (по 2 недели)

## Цель проекта

Создать онлайн-платформу, которая помогает проводить коллективные дегустации чая и собирать структурированные оценки участников.  
Платформа должна:

- позволять пользователям записываться на дегустации через Telegram-бота и/или веб;
- давать удобный инструмент для оценки чаёв (лучевые диаграммы);
- сохранять все данные на сервере для дальнейшего анализа и работы с ними (в том числе в DataLens);
- по итогам дегустации выдавать участникам простой, понятный отчёт и «профиль чаемана».

Работа разбита на три двухнедельных спринта. По результатам каждого спринта — рабочий, тестируемый функционал.

---

## Спринт 1 — Фундамент платформы

**Основная цель:** запустить скелет системы: регистрация через Telegram, базовая модель дегустаций, рабочий сервер в интернете.

### Что планируется сделать

1. **Архитектура системы**
   - Определение основных компонент:
     - сервис, обрабатывающий запросы от Telegram-бота и веб-интерфейса;
     - база данных для хранения пользователей, дегустаций и образцов чая;
     - простой веб-интерфейс-заглушка (страница «сервис работает»).
   - Описание сущностей: пользователи, дегустации, образцы чая.

2. **Регистрация и учёт пользователей через Telegram**
   - Подключение Telegram-бота к серверу.
   - При первом запуске бота пользователь автоматически заносится в систему (по его Telegram-аккаунту).
   - Фиксация основных данных: ID в Telegram, имя/ник.

3. **Создание основы для дегустаций**
   - Возможность завести дегустацию (название, дата, описание).
   - Привязка к дегустации списка чаёв (образцов).

4. **Развёртывание первого работающего прототипа**
   - Сервис развёрнут в интернете в виде Docker-приложения.
   - Есть отдельная выделенная база данных.
   - Базовые проверки доступности и работоспособности.

### Результат спринта 1

- Запущенный сервер, к которому подключён Telegram-бот.
- Пользователь, написавший боту, появляется в системе.
- Организатор может создать дегустацию и список чаёв (пока через внутренний интерфейс).
- Платформа готова к добавлению интерфейса ввода оценок.

## Спринт 2 — Флоу дегустации и интерфейс оценок

**Основная цель:** реализовать полный цикл оценки чаёв участниками с использованием лучевых диаграмм, а также связать Telegram и веб-интерфейс.

### Что планируется сделать

1. **Механика дегустации**
   - Для каждой дегустации задаётся набор характеристик оценки (например: аромат, сладость, горечь, послевкусие и т.д.).
   - Для каждого образца чая участник может выставить числовые оценки по этим характеристикам.
   - Все оценки сохраняются на сервере и привязаны к конкретному пользователю, дегустации и образцу чая.

2. **Связка Telegram → веб-интерфейс**
   - Пользователь получает от бота персональную ссылку на форму оценки (для конкретной дегустации/образца чая).
   - Переход по ссылке автоматически идентифицирует пользователя (без логина/пароля).
   - Веб-страница показывает форму с параметрами чая и возможностью ввести оценки.

3. **Интерфейс лучевых диаграмм**
   - На странице оценки отображается лучевая диаграмма (radar chart), которая визуализирует введённые пользователем значения.
   - При изменении оценок диаграмма обновляется.
   - Интерфейс адаптирован под использование с телефона (для дегустаций «из чайной»).

4. **Тестовая дегустация**
   - Проведение одной–двух тестовых дегустаций на небольшой группе.
   - Сбор первых реальных данных и обратной связи по удобству интерфейса.

### Результат спринта 2

- Участник дегустации:
  - записывается через Telegram-бота;
  - получает ссылку;
  - переходит на веб-страницу;
  - заполняет оценки по каждому чаю, видя визуальную диаграмму.
- Все данные аккуратно сохраняются и готовы к анализу.

## Спринт 3 — Отчёты для участников и админ-интерфейс

**Основная цель:** дать участнику понятный отчёт о его вкусовом профиле и сравнении с группой, а организатору — инструменты просмотра и выгрузки данных.

### Что планируется сделать

1. **Отчёт для участника**
   - Формирование профиля участника на основе его оценок:
     - какие характеристики он выделяет,
     - какие типы чаёв ему ближе.
   - Сравнение с группой:
     - лучевая диаграмма «участник vs средние оценки группы»;
     - текстовые выводы (например: «вы предпочитаете более сладкие и менее горькие чаи, чем средний участник этой дегустации»).
   - Ссылка на отчёт отправляется пользователю через Telegram по завершении дегустации.

2. **Интерфейс организатора (админка)**
   - Защищённый вход для организатора.
   - Просмотр списка дегустаций и их статусов.
   - Просмотр количества участников и завершённости заполнения анкет.
   - Выгрузка данных по дегустации в формате, удобном для аналитики (CSV/табличный формат).

3. **Подготовка к аналитике и DataLens**
   - Структура данных согласована с будущим подключением к Yandex DataLens.
   - Документация: какие таблицы и поля использовать для построения дашбордов.

### Результат спринта 3

- Участник получает персональный отчёт по результатам дегустации (диаграмма + короткий текстовый профиль).
- Организатор видит общую картину по дегустации и может скачивать данные для аналитики.
- Платформа готова к подключению инструментов BI (DataLens) и к дальнейшему развитию (ИИ-анализ, рекомендации, геймификация и т.п.).

## Дальнейшее развитие (после MVP)

После трёх спринтов можно развивать:

- более сложные профили и рекомендации по чаю;
- «историю дегустаций» для постоянных участников;
- интеграцию с ИИ-моделями для текстового анализа отзывов и генерации рекомендаций;
- продвинутые дашборды для организаторов и брендов.
